## 一、背景与目标

### 问题

开发者在日常工作中需要在浏览器中查看各种格式的原始文件（JSON 接口响应、本地 SQL 脚本、README.md、YAML 配置等）。当前没有一款扩展能统一处理所有常见格式，用户不得不同时安装多个独立扩展，且各扩展体验割裂、风格不一。

### 目标

打造一款 **Chrome 优先的统一多格式文件查看器扩展**，让开发者打开任何支持格式的文件时，自动获得一致、美观、高效的阅读体验，彻底告别"装一堆扩展"的困扰。

### 核心价值主张

> **一个扩展，搞定所有格式。本地处理，数据不出浏览器。**

---

## 二、目标用户

| 用户类型 | 痛点 | 使用场景 |
|---|---|---|
| 前后端开发者 | API 响应 JSON 难以阅读 | 调试接口、查看数据结构 |
| 运维 / DevOps | YAML/LOG 格式混乱 | 查看配置文件、排查日志 |
| 数据工程师 | SQL 脚本无高亮 | 查看数据库迁移脚本、CSV 文件 |
| 技术文档写作者 | 本地 Markdown 无预览 | 预览 README、文档草稿 |

**核心用户画像：** 经常需要在浏览器标签页中直接打开原始文件/接口的开发者，对效率和隐私有要求。

---

## 三、支持格式

首版（v1.0）覆盖以下 7 类格式：

| 格式 | 识别方式 | 渲染形态 |
|---|---|---|
| **JSON / JSONP** | Content-Type `application/json`、`.json` 后缀、内容嗅探 | 可折叠树视图 |
| **Markdown** | `.md` / `.markdown` 后缀、Content-Type `text/markdown` | HTML 渲染 |
| **SQL** | `.sql` 后缀 | 语法高亮代码块 |
| **YAML** | `.yaml` / `.yml` 后缀、Content-Type `application/yaml` | 可折叠树视图 |
| **XML** | Content-Type `text/xml` / `application/xml`、`.xml` 后缀 | 可折叠树视图 |
| **CSV** | `.csv` 后缀、Content-Type `text/csv` | 表格视图 |
| **LOG** | `.log` 后缀 | 行号 + 级别着色 |

## 四、功能需求

### 4.1 P0 — 核心功能（v1.0 必须上线）

#### 自动识别与渲染
- 用户打开符合条件的 URL 或本地文件时，扩展**自动接管页面**进行美化渲染，无需手动操作
- 识别失败时降级为原始文本，不报错、不白屏
- 提供 **Raw / 格式化** 一键切换，方便用户查看原始内容

#### JSON 渲染器
- 可折叠 / 展开的树视图，支持全部折叠 / 全部展开
- 语法高亮（字符串/数字/布尔/null/键名 分色）
- 数组和对象节点显示子项计数（如 `[42 items]`）
- URL 值自动变为可点击链接
- JSON Path 导航（点击节点显示路径，支持复制）
- 搜索 / 过滤（关键字高亮定位）
- JSONP 自动剥离回调函数包装

#### Markdown 渲染器
- GitHub Flavored Markdown（GFM）完整支持
- 代码块语法高亮（主流语言）
- 自动生成目录（ToC），侧边栏浮动
- Mermaid 流程图渲染
- 数学公式（KaTeX）
- 图片、表格、任务列表正常渲染
- 文件变动自动刷新（本地文件场景）

#### SQL 渲染器
- 关键字格式化（缩进、换行、大小写规范化）
- 语法高亮（关键字/函数/字符串/注释/表名分色）
- 多方言支持：MySQL、PostgreSQL、SQLite、BigQuery、MSSQL
- 关键字大小写可选（UPPER / lower / preserve）

#### YAML 渲染器
- 可折叠树视图（与 JSON 体验对齐）
- 语法高亮、锚点/引用高亮
- 内联注释保留显示

#### XML 渲染器
- 可折叠树视图
- 属性、文本节点、注释分色
- 长属性折叠

#### CSV 渲染器
- 表格视图，支持列宽拖拽
- 点击列头排序
- 行号显示
- 超大文件分页或虚拟滚动

#### LOG 渲染器
- 行号显示
- 日志级别着色：`ERROR`（红）/ `WARN`（黄）/ `INFO`（蓝）/ `DEBUG`（灰）
- 关键字搜索 + 过滤（仅显示匹配行）
- 时间戳识别并格式化

#### 通用能力
- **暗色 / 亮色模式**，跟随系统或手动切换
- **多主题**（不少于 5 套：GitHub Light / Dark / Dracula / Solarized / One Dark）
- **一键复制**（复制格式化内容 / 复制原始内容）
- **下载**格式化后的文件
- **字体大小**调节
- 本地文件支持（引导用户开启 `Allow access to file URLs` 权限）
- Popup 控制面板：开关各格式渲染、切换主题

### 4.2 P1 — 差异化功能（v1.1 迭代）

- **跨格式 Diff 对比**：粘贴两段内容，逐行高亮差异
- **格式互转**：JSON ↔ YAML ↔ CSV ↔ XML 一键转换
- **大文件虚拟滚动**：10 MB 以上文件流畅渲染，不卡顿
- **折叠记忆**：刷新后保留上次折叠状态

### 4.3 P2 — 增值功能（v2.0 规划）

- 导出为 HTML / PDF / Word
- SQL 执行计划可视化（EXPLAIN 结果解析）
- 自定义主题编辑器
- 多文件标签页管理

## 五、非功能需求

| 维度 | 要求 |
|---|---|
| **隐私** | 零网络请求，所有处理在浏览器本地完成；不收集任何用户数据 |
| **性能** | 1 MB 以内文件渲染 < 500ms；10 MB 文件不白屏（虚拟滚动） |
| **安全** | 禁止 `eval()`；HTML 渲染前 XSS 转义；CSP 策略严格配置 |
| **兼容性** | Chrome 最新 3 个主版本；Manifest V3 规范 |
| **包体积** | 扩展压缩包 < 2 MB（影响安装速度和审核时间） |

## 六、技术选型建议

| 模块 | 推荐方案 | 备注 |
|---|---|---|
| 扩展框架 | 原生 Manifest V3 | 无需框架，减少体积 |
| JSON/树视图 | 自研 + `json-tree` / `ztree` | 控制体积和定制化 |
| Markdown | `markdown-it` + `highlight.js` | 生态最完整，按需加载 |
| SQL 格式化 | `sql-formatter` | 支持多方言，轻量 |
| YAML | `js-yaml` | 标准库 |
| CSV | `papaparse` | 业界标准 |
| 数学公式 | `KaTeX`（非 MathJax） | 体积更小 |
| 图表 | `mermaid.js`（按需加载） | 仅 MD 场景用到 |
| UI 构建 | Vanilla JS / Preact | 避免引入 React 增加体积 |

## 七、用户体验关键路径

```
用户打开 .json URL
  → Content Script 检测 Content-Type / URL 后缀
  → 识别为 JSON 格式
  → 隐藏原始 <pre> 内容
  → 注入渲染容器，解析 JSON，渲染树视图
  → 工具栏显示（主题切换 / raw 切换 / 复制 / 下载）
  → 用户可交互（折叠、搜索、复制路径）
```

**本地文件首次使用流程：**
```
用户拖入本地 .md 文件
  → 扩展检测到无权限
  → 弹出引导提示（带截图说明如何开启 Allow access to file URLs）
  → 用户开启权限后自动刷新，正常渲染
```
